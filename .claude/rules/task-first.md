# Task-First Gate

Before writing or editing code, verify you have an active task in `specs/active/tasks.md` marked `[→]` (in-progress).

If no task is in-progress:

1. Check if an existing pending task `[ ]` matches the work — mark it `[→]`
2. If none matches, create a new task with proper format (T-number, description, status `[→]`)

Skip this for trivial fixes (typos, single-line changes) or non-code actions (research, reading, answering questions).
